WCAG 2.0

WCAG 2.0 includes two success criteria that are meant to give the developers the information to ensure the contrast between the content and its background is enough to ensure it is fully perceivable. The two criteria only differs in the thresholds for the contrast ratio and one is for the level AA and the other for AAA.

According to them, to ensure that the visual presentation of text (and images of text) is distinguishable, the contrast ratio (based on the font-size) should be at least:

For level AA - 1.4.3 Contrast (Minimum)
4.5 - font-size small
3 - font-size large
For level AAA - 1.4.6 Contrast (Enhanced)
7 - font-size small
4.5 - font-size large

According to the WCAG, small font is when the computed font-size for the text is smaller than 18 point or 14 point bold. Otherwise, it would be consider large.

The success criteria do not apply for incidental text or logotypes. Incidental text is the text (or images of text) that is part of an inactive user interface component, that is pure decoration, that is not visible to anyone, or that is part of a picture that contains significant other visual content.

WCAG Contrast Checker

It is a browser extension that analyses the contrast between the foreground and the background of all elements in the page. It identifies the computed colors and also considers the opacity of the background in order to determine the visual value. In addition to the contrast value, it also gets the computed font-size for the elements, so it can evaluate if the contrast is valid or not according to the WCAG 2.0 validation level (AA or AAA).

The extension does not consider images. This means that if the element is an image, the detected color for the foreground would be the one that its alternative text would have, and if if the element includes and image as background, the background color would be the one that it has defined without considering the image.

Visible and Hidden elements

Visible elements are the elements that do not use (or are not descendant of an element that uses) CSS properties "display" or "visibility" with values "none" or "hidden" respectively. Because there could be other ways for hiding elements, this list could include some that are also hidden or outside of the view-port area.

Hidden elements are the elements that use (or are descendant of an element that uses) the CSS properties "display" or "visibility" with values "none" or "hidden" respectively. These elements are usually part of collapsible menus, panels, fly-outs, etc. The page could still have other hidden elements that were not detected automatically. We should not consider these elements as incidental text since they could be shown under some circumstances like after user interaction.

Refresh on DOM updates

The analysis can be performed to be executed on every DOM modification (additions or remove of DOM elements) or on-demand. If it is executed automatically when de DOM is modified, it includes a debouncing time in order to prevent unneeded repetitions that could affect to the performance. When this feature is deactivated, it is possible to update the analysis manually by using the button "refresh".

Custom color analysis

The extension includes a color analyser that allows to select (or to introduce manually) a color combination to check the contrast difference between them and validate it according to different font sizes.
